# 🏭 SCM 협력사 변경 검토 대시보드 v7

> **SCM팀 외주생산 파트 전용** — 협력사 변경 검토 보고서를 작성·저장·관리하는 로컬 HTML 도구입니다.

---

## 📌 접속 방법

별도 설치나 인터넷 연결 없이 파일을 열면 바로 사용할 수 있습니다.

```
📁 프로젝트 폴더 > scm_dashboard_v7.html  ← 더블클릭으로 열기
```

> **Chrome 브라우저** 사용을 권장합니다 (PDF 출력 품질 최적화).

│  └────────────────────────────────────────────────┘  │

## 📝 보고서 작성 방법 (단계별)

### Step 1 — 새 검토 시작

상단 분기 탭 오른쪽의 **[+ 새 검토]** 버튼을 클릭하면 입력 폼이 열립니다.

---

### Step 2 — 아이템 선택 (3단계)

**① 굿즈명 입력**
- 검색창에 굿즈명 일부를 입력하면 자동완성 목록이 표시됩니다.
- 목록에서 원하는 굿즈를 클릭해 선택합니다.

**② 파츠유형 선택**
- 굿즈 선택 후 해당 굿즈의 파츠유형(칩 버튼)이 표시됩니다.
- 변경 검토 대상 파츠유형을 클릭해 선택합니다. (중복 선택 가능)

**③ 파츠명 선택**
- 파츠유형 선택 후 해당 파츠 목록이 표시됩니다.
- 검토 대상 파츠를 체크합니다.

> **파츠를 체크하는 순간 AS-IS 정보가 자동으로 채워집니다.**

---

### Step 3 — 전환 검토 배경 선택

| 항목 | 선택지 |
|------|--------|
| 대분류 | 품질 이슈 / 원가 이슈 / 납기 이슈 / 케파 부족 / 협력사 폐업 / 기타 |
| 세부사유 | 대분류 선택 시 연동 표시 |

---

### Step 4 — AS-IS 현황 확인

파츠 선택 시 아래 항목이 **자동 입력**됩니다. 실제 현황과 다를 경우 직접 수정 가능합니다.

| 항목 | 설명 |
|------|------|
| 현재 협력사명 | Airtable 파츠 데이터 기준 |
| 소재 | 파츠 지류/소재 |
| 임가공 | 발주 세부사항 기준 |
| MOQ | 굿즈별 최소 발주 수량 |
| 수량별 단가 | 1 / 50 / 100 / 300 / 500 / 1,000 / 3,000ea 기준 |
| 리드타임 | 100 / 300 / 500 / 1,000 / 3,000ea 기준 |
| 이슈 현황 | 최근 품질·납기·기타 이슈 자동 연결 |
| 판매량 추이 | 2025 / 2026년 월별 판매 수량 자동 연결 |

---

### Step 5 — TO-BE 정보 입력

변경 후 협력사 정보를 직접 입력합니다.

| 항목 | 입력 방법 |
|------|----------|
| 변경 협력사명 | 직접 텍스트 입력 |
| MOQ | 숫자 입력 |
| 샘플 발주 여부 | 드롭다운 선택 (아래 참고) |
| 비고 | 자유 입력 |
| 수량별 단가 | 각 수량별 직접 입력 (원 단위) |
| 리드타임 | 각 수량별 직접 입력 (일 단위) |

**샘플 발주 여부 선택지**

| 상태 | 의미 |
|------|------|
| 샘플 발주 완료 | 발주는 완료, 수령 전 |
| 샘플 발주 진행 중 | 발주 협의 진행 중 |
| 샘플 미발주 | 아직 발주 전 |
| 샘플 수령 완료 | 수령 완료, 테스트 전 |
| 결과 합격 | 샘플 테스트 합격 |
| 결과 불합격 | 샘플 테스트 불합격 |
| 샘플 재작업 필요 | 수정 후 재테스트 필요 |

---

### Step 6 — 비교 결과 확인

단가·리드타임을 입력하면 AS-IS 대비 차이가 자동 계산됩니다.

- 🟦 **파란색**: TOBE가 유리 (단가↓ / LT↓)
- 🟥 **빨간색**: TOBE가 불리 (단가↑ / LT↑)
- 수량별 단가 비교 차트가 자동으로 그려집니다.

---

### Step 7 — 전환 판단 선택

**GO / HOLD / NO** 중 하나를 선택하고 의견을 자유 텍스트로 작성합니다.

| 판단 | 의미 |
|------|------|
| ✅ GO | 전환 권고 |
| ⏸ HOLD | 추가 검토 필요 |
| ❌ NO | 전환 불가 |

---

### Step 8 — 저장 및 출력

| 버튼 | 기능 |
|------|------|
| 💾 **저장 및 아카이브** | 브라우저에 보고서 저장 + 분기 탭에 자동 분류 |
| 📄 **MD 다운로드** | 마크다운 파일 저장 (Notion·Obsidian 등 활용) |
| 🖨 **PDF 다운로드** | 단가 비교 차트·리드타임 포함 PDF 출력 |

## 🗃️ 아카이브 관리

저장된 보고서는 상단 **분기 탭**으로 구분되며, 탭 클릭 시 해당 분기 보고서만 표시됩니다.

각 보고서 카드에서 할 수 있는 작업:

| 버튼 | 기능 |
|------|------|
| 불러오기 | 저장된 보고서를 폼에 불러와 수정 |
| MD | 마크다운 파일 다운로드 |
| 삭제 | 해당 보고서 삭제 |

---

## 💾 보고서 백업 및 복원

아카이브 영역 상단의 백업 버튼으로 보고서 데이터를 PC에 백업하고 복원할 수 있습니다.

### 📤 내보내기 (백업)

```
[📤 내보내기] 버튼 클릭
→ SCM_보고서_백업_YYYYMMDD.json 파일 자동 다운로드
```

- 현재 저장된 **모든 보고서**를 JSON 파일로 내보냅니다.
- PC를 바꾸거나, 브라우저 캐시를 지우기 전에 반드시 백업하세요.

### 📥 가져오기 (복원)

```
[📥 가져오기] 버튼 클릭
→ 백업한 JSON 파일 선택
→ 기존 보고서에 자동 병합 (중복 제외)
```

- 백업 파일을 불러오면 기존 보고서와 **병합**됩니다 (덮어쓰기 아님).
- 다른 PC에서 작업한 보고서를 합칠 때도 활용 가능합니다.

### 🔄 자동 복원

캐시를 삭제해도 **대시보드 HTML 파일 자체에 마지막 빌드 시점의 보고서가 내장**되어 있어, 캐시 삭제 후 첫 접속 시 자동으로 복원됩니다.

> 단, 캐시 삭제 이후 **새로 작성한 보고서**는 복원되지 않으므로 정기적으로 📤 내보내기를 권장합니다.

## 🔄 데이터 자동 갱신

Airtable 파츠 데이터는 **매주 월요일 오전 8시(KST)** 에 자동으로 갱신됩니다.

헤더 배지에서 현재 데이터 기준일과 다음 갱신 예정일을 확인할 수 있습니다.

| 배지 | 의미 |
|------|------|
| 🟢 초록색 | 리드타임 데이터 정상 |
| 🟡 노란색 | 일부 파츠 리드타임 데이터 없음 |

> 데이터가 최신 현황과 다를 경우 AS-IS 항목을 직접 수정하거나 파트장에게 문의하세요.

---

## ⚠️ 주의사항

1. **Chrome 브라우저** 사용을 권장합니다.
2. 보고서는 **해당 기기의 브라우저에만 저장**됩니다. 다른 PC에서는 보이지 않습니다.  
   → 여러 기기에서 작업하는 경우 📤 내보내기 / 📥 가져오기를 활용하세요.
3. 브라우저 캐시 삭제 전 반드시 **[📤 내보내기]** 로 백업하세요.
4. AS-IS 단가·리드타임은 **Airtable 데이터 기준**이며, 실제 현황과 다를 수 있습니다.
5. 리드타임은 **Phantom 파츠(추상적 산출물) 기준**으로 표시됩니다.
6. PDF 출력 시 팝업 차단이 되어 있으면 출력되지 않습니다.  
   → 브라우저 팝업 허용 후 다시 시도하세요.

## 📂 파일 구조 (관리자용)

```
📁 프로젝트 폴더/
├── scm_dashboard_v7.html     ← 대시보드 본체 (파트원은 이 파일만 사용)
├── scripts/
│   ├── build_v7.py           ← HTML 빌더 (데이터 갱신 후 실행)
│   └── data_manager.py       ← Airtable 데이터 관리
└── data/
    ├── db_slim.json          ← 파츠·이슈·판매 데이터
    ├── goods_moq.json        ← 굿즈별 MOQ
    ├── items_lt_by_parts.json ← 파츠별 리드타임
    ├── parts_imgang.json     ← 파츠별 임가공 정보
    ├── reports_archive.json  ← 보고서 아카이브 (HTML 내장용)
    └── metadata.json         ← 버전·갱신일 정보
```

> **파트원은 `scm_dashboard_v7.html` 파일만 열면 됩니다.**  
> 데이터 갱신·파일 관리는 파트장이 담당합니다.
